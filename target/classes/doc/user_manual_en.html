Install and run FullSWOF-transfer_UI

You must have Java version 1.8 or higher to be able to run FullSWOF-transfer_UI.

The FullSWOF development team uses Java version 11 to verify the proper functioning of FullSWOF-transfer_UI. Thus, we guarantee its use with Java version 11.
To check the version of Java that is installed on your machine, type the following command in a terminal:

java -version

It is not necessary to have FullSWOF_1D to launch FullSWOF-transfer_UI.

To launch FullSWOF-transfer_UI from the command line, go to the folder containing FullSWOF-transfer_UI.jar (without unzipping the file) and run the command:

java -jar FullSWOF-transfer_UI.jar

On some systems, it is possible to launch the application by right-clicking on the jar archive, opening it with Java.


Preferences and help

When you have launched the interface using the command line above, the application opens. By default, the application language matches the language of your computer. It contains a horizontal menu bar. We find the Preferences menu where you can choose your language. Choose between French and English, then be sure to restart the application, otherwise the change of language will not be done.

Also in the horizontal menu bar, you will find a Help tab. You then have this interface user manual at your disposal, as well as information regarding FullSWOF-transfer_UI.


Fonctionnalities

Import a settings file

To reuse data from a file, you must use the command 'Open a parameters file'. This command will attempt to read data from a file formatted as the generated transfer.txt file.

Fill in the parameters

You can also start from scratch and create the transfer.txt file yourself. FullSWOF-transfer_UI allows you to easily specify the configuration parameters needed to launch FullSWOF. Just fill in the different fields. Some fields use integer values, and others decimals.

For some fields, you are prompted for a file name. Select the browse button '...', then choose the appropriate file for that field.

Also, when you switch tabs and then return to the previous tab, the settings are not lost. You are free to fill in the fields in the order you wish.

Error management

When a field is not filled, it appears in red. You can then fill it, the red color will disappear.

If a field is incorrectly filled (for example, if a field expects an integer and you fill this field with a character string) then this field will indicate an error with the color red.

If you try to generate the file transfer.txt while there are errors or fields not filled, the application will indicate an error and will not generate the file.

File generation

If all the fields are filled in correctly, you will find a tab in the menu bar File, where there is a button Generate. The file is generated; warning, if the file already exists, it will be overwritten (you will see a warning message).
